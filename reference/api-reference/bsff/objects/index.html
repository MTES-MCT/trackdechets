Bsff​

Fields

NameDescription
id
ID!

Identifiant unique assigné par Trackdéchets. Il est à utiliser pour les échanges avec l'API.

createdAt
DateTime!

Date de création

updatedAt
DateTime!

Date de dernière modification

isDraft
Boolean!

Si ce BSFF est à l'état de brouillon ou pas.

Il y a principalement deux différences entre un brouillon et un BSFF publié :

  • Il n'y a pas de champs requis sur un brouillon, là où il faut un minimum d'informations pour créer un BSFF publié.
  • L'état de brouillon peut permettre de distinguer un BSFF prêt à circuler d'un BSFF encore en préparation. Par exemple, sur l'interface Trackdéchets les brouillons sont dans l'onglet "brouillon" et non pas "pour action" ou "à collecter".
type
BsffType!

Type de BSFF, voir l'enum pour plus de détails.

status
BsffStatus!

Statut qui synthétise où en est le déchet dans son cheminement, voir l'enum pour plus de détails.

emitter
BsffEmitter

Émetteur du déchet, qui n'est pas nécessairement le producteur. Il s'agit par exemple de l'opérateur ayant collecté des fluides lors d'interventions, ou alors d'une installation de collecte qui procède à la réexpédition pour traitement final.

packagings
[BsffPackaging!]!

Liste des contenants utilisés pour le transport du déchet.

waste
BsffWaste

Détails du déchet.

weight
BsffWeight

Quantité totale du déchet en kilogrammes, il peut s'agir d'une estimation.

transporter
BsffTransporter

Transporteur du déchet, effectue l'enlèvement du déchet auprès de l'émetteur et l'emporte à la destination.

À noter que l'émetteur peut également être transporteur, par exemple dans le cas de l'opérateur qui dépose lui même ses contenants auprès d'une installation de collecte. Dans ce cas il nous faut quand même savoir qui a effectué le transport, et indiquer l'opérateur à la fois en tant qu'émetteur et transporteur.

destination
BsffDestination

Destination du déchet, qui peut le réceptionner pour traitement, groupement, reconditionnement ou réexpedition. Un nouveau bordereau doit être créé si le déchet connaît un nouveau parcours, par exemple s'il quitte une installation de collecte pour un centre de traitement.

Pour plus de détails sur les différents types de bordereau, voir l'enum BsffType.

ficheInterventions
[BsffFicheIntervention!]!

Liste des fiches d'intervention associés à ce bordereau. Habituellement renseigné par un opérateur lors de son intervention.

forwarding
InitialBsff

Bordereau que celui-ci réexpédie

forwardedIn
Bsff

Bordereau dans lequel celui-ci est réexpédié

repackaging
[InitialBsff!]!

Bordereaux que celui-ci reconditionne

repackagedIn
Bsff

Bordereau dans lequel celui-ci est reconditionné

grouping
[InitialBsff!]!

Bordereaux que celui-ci groupe

groupedIn
Bsff

Bordereau dans lequel celui-ci est groupé.
