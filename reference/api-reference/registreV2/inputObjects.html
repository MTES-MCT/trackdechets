IncomingTexsLineInput​

Contenu d'une ligne pour l'import de terres excavées et sédiments, dangereux et non dangereux entrants

Arguments

NameDescription

reason
RegistryLineReason

Motif Indique si la ligne doit être modifiée ou supprimée. A laisser vide si c'est une création.

publicId
ID!

Identifiant unique Identifiant unique donné à chaque déclaration pour un établissement donné

Si un import comporte plusieurs mêmes identifiants uniques sans motif, seule la première ligne sera déclarée, les suivantes seront ignorées.

reportAsCompanySiret
String

SIRET du déclarant À compléter dans le cadre d'une délégation, SIRET de l'établissement qui fait la déclaration qui n'est pas celui pour qui la déclaration est faite.

Le SIRET renseigné doit être rattaché à mon compte Trackdéchets (faire partie de mes établissements) et doit faire l'objet d'une délégation active entre le SIRET de l'émetteur et mon SIRET.

reportForCompanySiret
String!

SIRET du destinataire Établissement pour lequel la déclaration est effectuée, soit ici le destinataire des terres et sédiments ou des déchets entrants

Le SIRET renseigné doit :

  • être inscrit sur Trackdéchets,
  • être rattaché à mon compte Trackdéchets (sauf délégation),
  • avoir au moins un des profils ou sous-profils suivants : Installation de valorisation de terres et sédiments et/ou Installation dans laquelle les déchets perdent leur statut de déchet et/ou : Installation de valorisation de terres et sédiments : Installation de Tri, transit regroupement de déchets y compris non classée Déchets non Dangereux (Rubriques 2713, 2714, 2715, 2716) Déchets Dangereux (Rubrique 2718) Autres cas déchets dangereux (Rubriques 2719, 2792-1, 2793-1, 2793-2, 2797-1, 2798) Installation de traitement : Incinération de déchets dangereux (Rubrique 2770) Incinération de déchets non dangereux (Rubriques 2771, 2740) stallation de stockage de déchets dangereux (Rubriques 2720-1, 2760-1, 2760-4, 2797-2) Installation de stockage de déchets non dangereux, y compris casiers dédiés amiante, plâtre (Rubriques 2720-2, 2760-2-a, 2760-2-b) Installation de stockage de déchets inertes (Rubrique 2760-3) Autres traitements de déchets non dangereux (Rubriques 2791, 2781, 2782, 2780, 2795) Autres traitements de déchets dangereux (Rubriques 2790, 2792-2, 2793-3)
  • faire l'objet d'une délégation active avec le SIRET du déclarant (dans le cas d'une délégation). La raison sociale ainsi que l'adresse de ce SIRET seront récupérés depuis la base Trackdéchets.

Dénomination usuelle des terres et sédiments ou des déchets

String

Code déchet

Si un code déchet est mentionné, il doit faire partie de la liste des codes déchets suivante : 17 05 03*, 17 05 04, 17 05 05*, 17 05 06, 20 02 02

wasteCodeBale
String

Code déchet Bâle

Le code déchet mentionné doit faire partie de la liste des codes déchets définis dans la Convention de Bâle

wastePop
Boolean!

POP Ce champ permet d'indiquer si les terres et sédiments ou les déchets entrants contiennent des polluants organiques persistants au regard de la définition de l'article R. 541-8 du code de l'environnement.

wasteIsDangerous
Boolean

Dangereux Indique si le déchet avec un code sans * est qualifié de dangereux

Si le code déchet est non dangereux et ne dispose pas de code miroir dangereux, il est possible de le forcer comme dangereux. Si aucune valeur n'est ajoutée, c'est les valeurs du code déchet et POP qui permettent d'autocompléter le caractère dangereux ou non dangereux du déchet entrant.

receptionDate
DateTime!

Date de réception Date de réception des terres et sédiments ou des déchets entrants

La date de réception ne peut pas :

  • être dans le futur,
  • être antérieure à J-1 an.

weightValue
Float!

Poids en tonnes Poids en tonnes des terres et sédiments ou des déchets entrants

La valeur doit être strictement comprise entre 0 et 1000, le séparateur de décimales peut être le point ou la virgule et le poids ne peut avoir plus de 3 décimales.

weightIsEstimate
Boolean!

Type de poids Indique si le poids est estimé ou réel. Si le code d'opération est R1 ou D10 ou D5, le poids doit obligatoirement être réel.

volume
Float

Volume en M3

La valeur doit être strictement comprise entre 0 et 1000, le séparateur de décimales peut être le point ou la virgule et le volume ne peut avoir plus de 3 décimales.

parcelInseeCodes
[String!]

Codes INSEE des parcelles Le ou les codes INSEE des communes des parcelles cadastrales du site de production, plusieurs numéros peuvent être renseignés

Chaque code INSEE doit faire 5 caractères, si plusieurs codes INSEE sont renseignés, ils doivent être séparés par virgules. Il est obligatoire de renseigner : soit les codes INSEE des communes des parcelles ainsi que leur(s) numéro(s) soit leurs coordonnées (identification précise du lieu géographique de production).

parcelNumbers
[String!]

Numéro(s) des parcelles Le ou les numéros des parcelles cadastrales du site de production, numéro composé d'un préfixe, d'une section et d'un numéro de parcelle

Chaque numéro de parcelle doit être composé d'un préfixe, d'une section et d'un numéro de parcelle avec préfixe : 3 chiffres, section : deux lettres et préfixe : 2 chiffres Plusieurs numéros peuvent être renseignés et doivent être séparés par des virgules Il est obligatoire de renseigner : soit les codes INSEE des communes des parcelles ainsi que leur(s) numéro(s) soit leurs coordonnées (identification précise du lieu géographique de production).

parcelCoordinates
[String!]

Coordonnées des parcelles Coordonnées de la ou des parcelles cadastrales du site de production par exemple : 48.894258 2.240027

Il est obligatoire de renseigner : soit les codes INSEE des communes des parcelles ainsi que leur(s) numéro(s) soit leurs coordonnées (identification précise du lieu géographique de production). Attention à bien séparer par des virgules des coordonnées de parcelles différentes, mais à bien utiliser le point comme séparateur décimal au sein même des coordonnées. Concernant le format de chaque coordonnées :

  • valeur attendue en degrés
  • le séparateur de décimales utilisé doit être un point
  • entre la latitude et la longitude d'une coordonnée, il doit y avoir un espace
  • pour séparer plusieurs coordonnées, une virgule doit être utilisée

Le numéro de notification (GISTRID) prévue à l'annexe I-B du règlement (CE) 1013/2006 pour un registre entrant au format PP AAAA DDDRRR avec PP pour le code pays, AAAA pour l'année du dossier, DDD pour le département de départ et RRR pour le numéro d'ordre; ou renseigner ici le numéro de déclaration Annexe 7 (optionnel) prévu à l'annexe 1-B du règlement N°1013/2006, au format A7E AAAA DDDRRR avec A7E pour Annexe 7 Export, AAAA pour l'année du dossier, DDD pour le département de départ et RRR pour le numéro d'ordre.
